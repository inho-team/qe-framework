# REMEDIATION_REQUEST Format

> Standard format for remediation request documents. Generated by Esupervision-orchestrator when a supervision result is rated FAIL.

## Storage Path

```
.qe/tasks/remediation/REMEDIATION_REQUEST_{UUID}_{N}.md
```

- `{UUID}`: Original TASK_REQUEST UUID
- `{N}`: Supervision round number (1, 2, 3)

---

## Document Format

```markdown
# REMEDIATION_REQUEST

## Header

| Field | Value |
|-------|-------|
| **Original UUID** | {UUID} |
| **Task Name** | {Original TASK_REQUEST task name} |
| **Supervision Round** | {N}/3 |
| **Created** | {YYYY-MM-DD HH:MM:SS} |

---

## Supervision Result

| Field | Value |
|-------|-------|
| **Overall Grade** | FAIL |
| **Supervision Agent(s)** | {List of supervision agents, comma-separated} |
| **Supervision Date** | {YYYY-MM-DD HH:MM:SS} |

### Domain Results

| Domain | Agent | Grade | Findings |
|--------|-------|-------|----------|
| {domain} | {agent name} | FAIL/PARTIAL/PASS | {N} issues |
| ... | ... | ... | ... |

---

## Deficient Items

### Item 1: {Item title}

- **Domain:** {Supervision domain}
- **Grade:** FAIL
- **Problem:** {Specific problem — what is deficient}
- **Remediation Directive:** {Remediation instruction — how it must be fixed}
- **Fix Direction:** {Specific fix direction — which part of the code/document to change and how}
- **Affected Files:** {List of related file paths}

### Item 2: {Item title}

- **Domain:** {Supervision domain}
- **Grade:** FAIL
- **Problem:** {Specific problem}
- **Remediation Directive:** {Remediation instruction}
- **Fix Direction:** {Specific fix direction}
- **Affected Files:** {List of related file paths}

(Repeat for each FAIL item)

---

## Remediation Checklist

> Composed as specific directives targeting only deficient items. Etask-executor executes this checklist in order.

- [ ] {Specific remediation directive for Item 1}
- [ ] {Specific remediation directive for Item 2}
- [ ] {Re-verification directive for items affected by the fix}

---

## Round Information

| Field | Value |
|-------|-------|
| **Current Round** | {N}/3 |
| **Previous Rounds** | {File paths of previous round REMEDIATION_REQUESTs, or "N/A"} |
| **Remaining Rounds** | {3 - N} |
| **Escalation** | {If N == 3: "Next failure triggers user escalation", otherwise: "N/A"} |
```

---

## Field Descriptions

| Field | Description |
|-------|-------------|
| **Original UUID** | UUID shared between TASK_REQUEST and VERIFY_CHECKLIST |
| **Supervision Round** | Current supervision-remediation loop iteration. Maximum 3 rounds |
| **Overall Grade** | REMEDIATION_REQUEST is only generated when the grade is FAIL |
| **Problem** | Specific issue found by the supervision agent — "what is insufficient" |
| **Remediation Directive** | Instruction delivered to the remediation executor — "what must be done" |
| **Fix Direction** | Specific direction of the fix — "where and how to change it" |
| **Remediation Checklist** | Checklist-format directives to be executed by Etask-executor |

## Rules

- REMEDIATION_REQUEST is generated only when the supervision grade is **FAIL**
- PARTIAL grade is a conditional pass — REMEDIATION_REQUEST is not generated
- Each round's REMEDIATION_REQUEST is saved as an independent file and does not overwrite previous rounds
- The Remediation Checklist includes only FAIL items (PASS/PARTIAL items are excluded)
- If FAIL persists after round 3, Esupervision-orchestrator escalates to the user
