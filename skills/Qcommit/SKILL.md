# Qcommit — 자연스러운 커밋

## 역할
AI가 작성했다는 흔적 없이, 사람이 직접 작성한 것처럼 자연스러운 git 커밋을 생성합니다.
실제 커밋 작업은 `Ecommit-executor` 서브에이전트에게 위임합니다.

## 금지 사항 (절대 규칙)
- `Co-Authored-By` 라인 **절대 추가하지 않음**
- `Generated by`, `Created by AI`, `Claude`, `GPT` 등 AI 관련 문구 금지
- 이모지 사용 금지
- "체계적으로", "종합적으로", "효율적으로" 같은 AI 문체 금지

## 커밋 메시지 규칙

### 제목 (첫 줄)
- 소문자 시작, 70자 이내
- Conventional Commits 형식: `type: 설명`
- type: `feat`, `fix`, `refactor`, `chore`, `docs`, `test`, `style`, `perf`
- 간결하고 구체적으로 — "뭘 했는지"만

### 본문 (선택)
- 빈 줄로 구분
- "왜" 변경했는지 간략히
- 불필요하면 생략

### 예시
```
feat: 사용자 인증 API 추가

로그인/회원가입 엔드포인트 구현.
JWT 기반 토큰 발급.
```

```
fix: 주문 금액 계산 오류 수정
```

```
refactor: DB 커넥션 풀 설정 분리
```

## 실행 절차

### 1단계: 변경사항 확인
- `git status`로 변경된 파일 확인
- `git diff`로 변경 내용 파악

### 2단계: 커밋 메시지 작성
- 변경 내용을 분석하여 자연스러운 커밋 메시지 작성
- 프로젝트의 기존 커밋 스타일 확인 (`git log --oneline -10`)
- 기존 스타일에 맞춤

### 3단계: 스테이징 및 커밋
- 관련 파일만 선택적으로 `git add` (git add -A 지양)
- `.env`, credentials 등 민감 파일 제외 확인
- 커밋 실행

### 4단계: 결과 표시
- 커밋 해시, 변경 파일 수 간략히 표시

## 할 것 (Will)
- 변경사항 분석 및 자연스러운 커밋 메시지 작성
- 프로젝트 커밋 스타일에 맞춤
- 민감 파일 스테이징 방지

## 안 할 것 (Will Not)
- Co-Authored-By 추가
- AI 관련 문구/흔적 포함
- git push (사용자가 명시적으로 요청하지 않는 한)
- 변경사항 없을 때 빈 커밋 생성
