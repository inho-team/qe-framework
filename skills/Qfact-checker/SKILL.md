# Fact-Checker — 주장 검증 워크플로우

문서, 보고서, 기사에서 사실적 주장을 추출하고 체계적으로 검증한다.

## 검증 프로세스

```
1. 주장 추출 → 2. 우선순위 분류 → 3. 증거 수집 → 4. 출처 확인 → 5. 판정 → 6. 보고서 작성
```

## Step 1: 주장 추출

### 검증 대상

| 대상 | 예시 |
|------|------|
| 사실적 주장 | "X가 발생했다", "Y는 사실이다" |
| 통계/수치 | "40% 증가", "100만 명" |
| 날짜/시간 | "2024년 1월에 시작" |
| 인용/귀속 | "A가 말했다" |
| 인과 주장 | "X 때문에 Y가 발생했다" |

### 검증 제외

- 의견: "이 정책은 좋다/나쁘다"
- 예측: "미래에 X가 될 것이다"
- 취향: "A보다 B가 낫다"

### 주장 추출 템플릿

```markdown
## 주장 로그

**문서:** [출처]
**날짜:** [작성일]

### 주장 1
- **진술:** [정확한 인용 또는 요약]
- **화자:** [누가 말했는지]
- **맥락:** [주변 문맥]
- **유형:** 통계 / 역사적 사실 / 인용 / 인과
- **우선순위:** 높음 / 중간 / 낮음
- **상태:** 미검증 / 검증됨 / 거짓 / 검증불가
```

### 우선순위 기준

| 우선순위 | 기준 |
|----------|------|
| **높음** | 문서의 핵심 논지, 쉽게 검증 가능, 오류 시 영향 큼 |
| **중간** | 보조 근거, 검증에 노력 필요 |
| **낮음** | 주변 디테일, 통상적으로 수용, 영향 미미 |

## Step 2: 증거 수집

### 출처 우선순위

| 주장 유형 | 1차 출처 |
|-----------|----------|
| 통계 | 원본 연구, 정부 데이터, 조사 방법론 |
| 인용 | 음성/영상 녹음, 속기록, 직접 확인 |
| 역사 | 당시 뉴스 보도, 공식 기록 |
| 과학 | 피어리뷰 논문, 전문가 합의 |
| 법률 | 법원 문서, 공식 제출 서류 |
| 재무 | 감사보고서, 공시 자료 |

### 증거 강도

| 증거 유형 | 강도 |
|-----------|------|
| 공식 문서 (법원 기록, 정부 보고서) | 강함 |
| 원본 데이터 (직접 분석 가능) | 강함 |
| 전문가 합의 (독립적 복수 전문가) | 강함 |
| 실명 출처 (직접 지식 보유) | 중간 |
| 당시 보도 (동시대 뉴스) | 중간 |
| 익명 출처 | 약함 |
| SNS 게시물 | 약함 |

## Step 3: 판정

### 판정 등급

| 등급 | 기준 |
|------|------|
| **사실** | 정확하고 완전함, 중요한 누락 없음 |
| **대체로 사실** | 정확하나 맥락 보충 또는 사소한 명확화 필요 |
| **절반 사실** | 부분적 정확, 핵심 맥락 누락 |
| **대체로 거짓** | 일부 사실 포함하나 전체적으로 오도 |
| **거짓** | 증거와 모순 |
| **검증불가** | 충분한 증거 없음 |

### 판정 템플릿

```markdown
## 판정: [주장 요약]

**주장:** [정확한 진술]
**화자:** [누가]
**판정:** [등급]

### 지지 증거
- [증거 1 + 출처]
- [증거 2 + 출처]

### 반박 증거
- [증거 1 + 출처]

### 누락된 맥락
- [맥락 1]

### 판정 근거
[왜 이 등급인지 설명]

### 신뢰도
높음 / 중간 / 낮음 + 이유
```

## Step 4: 보고서 작성

### 팩트체크 보고서 형식

```markdown
# 팩트체크 보고서

**대상 문서:** [제목/URL]
**검증일:** [날짜]
**검증자:** [이름]

## 요약
- 총 주장 수: N개
- 사실: N개 | 대체로 사실: N개 | 거짓: N개 | 검증불가: N개

## 주요 발견

### [주장 1] — 판정: [등급]
- **원문:** "[인용]"
- **증거:** [근거 요약]
- **수정 제안:** [필요 시]

### [주장 2] — 판정: [등급]
...

## 정정 필요 사항
| 위치 | 원문 | 수정안 | 사유 |
|------|------|--------|------|
| [섹션/줄] | [원문] | [수정] | [근거] |

## 출처 목록
1. [출처 1]
2. [출처 2]
```

## 실행 규칙

### MUST DO
- WebSearch 도구로 주장의 사실 여부를 검증한다
- 1차 출처(공식 문서, 원본 데이터)를 우선 확인한다
- 반박 증거도 적극적으로 검색한다
- 판정에는 반드시 근거를 명시한다
- 검증 불가한 주장은 "검증불가"로 정직하게 표시한다

### MUST NOT DO
- 증거 없이 "사실"로 판정하지 않는다
- 의견을 사실적 주장으로 분류하지 않는다
- 단일 출처만으로 "사실" 판정하지 않는다 (가능하면 복수 출처)
- 검증 과정을 생략하고 직관으로 판정하지 않는다
