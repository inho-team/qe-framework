# Qcompact — 맥락 보존

## 역할
컨텍스트 윈도우가 압박받기 전에 현재 작업 맥락을 `.qe/context/`에 저장하는 스킬.
compaction으로 잃어버릴 수 있는 핵심 정보를 보존합니다.

## 동작 방식

### 자동 실행 (기본)
- Ecompact-executor가 컨텍스트 압박을 감지하면 백그라운드에서 자동 실행
- 사용자에게 알리지 않고 조용히 저장
- MODE_TokenEfficiency의 Yellow 존(75%+) 진입 시 트리거

### 수동 실행
- `/Qcompact`로 직접 호출 가능
- 저장된 맥락 요약을 보여줌

## 저장 위치
`.qe/context/`

## 저장 내용

### `snapshot.md` — 맥락 스냅샷
매 저장 시 덮어씁니다 (최신 상태만 유지):

```markdown
# Context Snapshot
> 저장 시각: 2026-03-14 10:30

## 현재 작업
- 진행 중인 태스크 UUID 및 제목
- 체크리스트 진행 상황 (완료/전체)
- 현재 작업 중인 체크리스트 항목

## 핵심 결정사항
- 이번 세션에서 내린 주요 결정들
- 사용자가 명시적으로 지시한 방향

## 변경된 파일
- 이번 세션에서 생성/수정/삭제한 파일 목록

## 미완료 사항
- 아직 끝나지 않은 작업
- 다음에 해야 할 것

## 주의사항
- 특별히 기억해야 할 제약조건이나 요구사항
```

### `decisions.md` — 누적 결정 이력
세션별로 누적됩니다 (역순, 최신이 위):

```markdown
## [2026-03-14] 세션
- .qe/ 폴더로 프레임워크 데이터 분리 결정
- 에이전트 접두사 A → E 변경
- QE 프레임워크(Query Executor)로 브랜딩
```

## 할 것 (Will)
- 현재 작업 맥락 저장 (.qe/context/snapshot.md)
- 결정사항 누적 기록 (.qe/context/decisions.md)
- 변경 파일 목록 기록
- 미완료 사항 기록

## 안 할 것 (Will Not)
- 전체 대화 내용 저장 (핵심만 추출)
- 코드 전체 복사 (파일 경로만 기록)
- 사용자에게 알림 (자동 실행 시)
